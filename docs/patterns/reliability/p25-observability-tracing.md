# P25 · OBSERVABILITY-TRACING — オブザーバビリティ / トレース＋プロベナンス

!!! abstract "一言で"
    推論ステップ・ツール呼び出し・トークン数・コスト・レイテンシ・評価スコアを構造化トレースで記録するパターンです。トレースIDで全工程を一貫して辿れるようにし、確率的な挙動のデバッグ・コスト按分・品質劣化検知・監査を可能にします。さらに、規制下の重要な意思決定については「なぜその結論に至ったか」を参照データ・推論経路・使用モデル・プロンプトバージョン・承認者まで遡れるプロベナンス（来歴）として記録し、説明責任に耐える証跡を残します。意思決定の設計判断は人間のレビューが必須です。

## 解決する問題

- **確率的挙動のデバッグ**: 「なぜこの回答になったか」を再現・分析できません。
- **コスト按分**: 部署・プロジェクト・エージェント単位のコストを把握できません。
- **品質劣化検知**: モデル変更やプロンプト変更による品質劣化をサイレントに見逃してしまいます。
- **監査**: 「誰が・いつ・何を・どのモデルで」を追跡できません。
- **規制対応**: 金融・人事・与信等の規制下で、意思決定プロセスの説明が義務付けられています。
- **説明責任**: 「なぜこの判断になったか」を後から説明できません。
- **紛争時の証拠**: 顧客や従業員からの異議申し立てに対し、根拠を示せません。
- **バイアス監査**: 意思決定にバイアスがないか検証するためのデータが不足しています。

## 選定条件

**採用する場合（When to use）**

- 本番運用するすべてのエージェント（例外なし）。
- 規制下の意思決定（金融/人事/与信/保険/公共）を行う場合は、プロベナンス拡張を追加してください。
- 説明責任・異議申し立て対応が求められる業務。

**採用しない場合（When NOT）**

- 特にありません（本番では必須パターンです）。プロベナンス拡張は低リスク・娯楽的な用途では不要です。

## 構造

- OpenTelemetry GenAI semantic conventions 準拠のスパンで、推論・ツール呼び出し・検索・メモリアクセスの各ステップを記録します。
- トレースIDで全工程を辿れるようにし、分散システム全体の因果関係を可視化します。
- **ログ基盤にはメタデータ**（ID・モデル名・トークン数・レイテンシ・コスト・評価スコア・ツール名・成否・要約）を格納し、**オブジェクトストレージにはプロンプト本文・コンテキスト・生出力**を格納します（IDで紐付けます）。
- 正常系はサンプリング、エラー/低評価/高コストは全量記録します（tail-based sampling）。
- PIIはマスキングし、保持期間はデータ分類別に設定します。

### 規制対応のプロベナンス拡張

- トレースを拡張し、最終出力から根拠文書・ツール結果・使用モデル・プロンプトバージョン・承認者（[P18](../orchestration/p18-hitl-approval-gate.md)）を一貫して逆引き可能にします。
- 決定ログを不変の監査ログ（append-only）として保持します。
- 規制説明に耐える粒度で、因果関係を辿れる形式で記録します。
- 根拠リンク付き生成（grounded generation）により、出力を参照元ドキュメントまで追跡可能にします。

## 駆動変数とチューニング（程度）

全プロンプト本文をログ基盤に入れるのは容量・コスト面で非現実的です。ログ基盤にはメタデータのみを格納し、本文はオブジェクトストレージに分離して、IDで紐付けましょう。正常系はサンプリング（目安として1〜10%を起点）、エラー・低評価・高コストのリクエストは全量記録します（tail-based sampling）。PIIはマスキングし、保持期間はデータ分類に応じて設定してください。

## ユースケース

- 全社AIゲートウェイで全推論リクエストのメタデータ（モデル・トークン・レイテンシ・コスト）をOpenTelemetryスパンとしてLangfuseに送信し、ダッシュボードで部署別コスト・品質スコアの推移を監視します。
- Salesforceの営業エージェントで低評価の回答が発生した際、トレースIDから推論経路・参照したドキュメント・ツール呼び出し結果を辿り、原因を特定します。
- 金融機関の与信審査エージェントが「融資不可」と判定した際、参照した信用情報・使用モデル・プロンプトバージョン・判定ロジック・承認者を不変ログに記録し、顧客からの異議申し立て時に根拠を提示します。
- 人事エージェントが採用候補者のスクリーニングを行った際、判定に使用した情報と推論経路を記録し、バイアス監査に活用します。

## 実装メモ

OpenTelemetry GenAI semantic conventions、Langfuse / LangSmith / Arize、オブジェクトストレージ（S3等）、tail-based sampling、根拠リンク付き生成（grounded generation）、決定ログ（append-only）、不変監査ログ、モデル/プロンプトのバージョン記録。
