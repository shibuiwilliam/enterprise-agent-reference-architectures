# P25 · OBSERVABILITY-TRACING — オブザーバビリティ / トレース＋プロベナンス

!!! abstract "一言で"
    推論ステップ・ツール呼び出し・トークン数・コスト・レイテンシ・評価スコアを構造化トレースで記録するパターン。トレースIDで全工程を一貫して辿れるようにし、確率的な挙動のデバッグ・コスト按分・品質劣化検知・監査を可能にする。さらに、規制下の重要な意思決定については「なぜその結論に至ったか」を参照データ・推論経路・使用モデル・プロンプトバージョン・承認者まで遡れるプロベナンス（来歴）として記録し、説明責任に耐える証跡を残す。意思決定の設計判断は人間のレビューが必須。

## 解決する問題

- **確率的挙動のデバッグ**: 「なぜこの回答になったか」を再現・分析できない。
- **コスト按分**: 部署・プロジェクト・エージェント単位のコストを把握できない。
- **品質劣化検知**: モデル変更やプロンプト変更による品質劣化をサイレントに見逃す。
- **監査**: 「誰が・いつ・何を・どのモデルで」を追跡できない。
- **規制対応**: 金融・人事・与信等の規制下で、意思決定プロセスの説明が義務付けられている。
- **説明責任**: 「なぜこの判断になったか」を後から説明できない。
- **紛争時の証拠**: 顧客や従業員からの異議申し立てに対し、根拠を示せない。
- **バイアス監査**: 意思決定にバイアスがないか検証するためのデータが不足している。

## 選定条件

**採用する場合（When to use）**

- 本番運用するすべてのエージェント（例外なし）。
- 規制下の意思決定（金融/人事/与信/保険/公共）を行う場合は、プロベナンス拡張を追加する。
- 説明責任・異議申し立て対応が求められる業務。

**採用しない場合（When NOT）**

- 特になし（本番では必須パターン）。プロベナンス拡張は低リスク・娯楽的な用途では不要。

## 構造

- OpenTelemetry GenAI semantic conventions 準拠のスパンで、推論・ツール呼び出し・検索・メモリアクセスの各ステップを記録する。
- トレースIDで全工程を辿れるようにし、分散システム全体の因果関係を可視化する。
- **ログ基盤にはメタデータ**（ID・モデル名・トークン数・レイテンシ・コスト・評価スコア・ツール名・成否・要約）を格納し、**オブジェクトストレージにはプロンプト本文・コンテキスト・生出力**を格納する（IDで紐付け）。
- 正常系はサンプリング、エラー/低評価/高コストは全量記録する（tail-based sampling）。
- PIIはマスキングし、保持期間はデータ分類別に設定する。

### 規制対応のプロベナンス拡張

- トレースを拡張し、最終出力から根拠文書・ツール結果・使用モデル・プロンプトバージョン・承認者（[P18](../orchestration/p18-hitl-approval-gate.md)）を一貫して逆引き可能にする。
- 決定ログを不変の監査ログ（append-only）として保持する。
- 規制説明に耐える粒度で、因果関係を辿れる形式で記録する。
- 根拠リンク付き生成（grounded generation）により、出力を参照元ドキュメントまで追跡可能にする。

## 駆動変数とチューニング（程度）

全プロンプト本文をログ基盤に入れるのは容量・コスト面で非現実的。ログ基盤にはメタデータのみ、本文はオブジェクトストレージに分離し、IDで紐付ける。正常系はサンプリング（目安として1〜10%を起点）、エラー・低評価・高コストのリクエストは全量記録する（tail-based sampling）。PIIはマスキングし、保持期間はデータ分類に応じて設定する。

## ユースケース

- 全社AIゲートウェイで全推論リクエストのメタデータ（モデル・トークン・レイテンシ・コスト）をOpenTelemetryスパンとしてLangfuseに送信し、ダッシュボードで部署別コスト・品質スコアの推移を監視する。
- Salesforceの営業エージェントで低評価の回答が発生した際、トレースIDから推論経路・参照したドキュメント・ツール呼び出し結果を辿り、原因を特定する。
- 金融機関の与信審査エージェントが「融資不可」と判定した際、参照した信用情報・使用モデル・プロンプトバージョン・判定ロジック・承認者を不変ログに記録し、顧客からの異議申し立て時に根拠を提示する。
- 人事エージェントが採用候補者のスクリーニングを行った際、判定に使用した情報と推論経路を記録し、バイアス監査に活用する。

## 実装メモ

OpenTelemetry GenAI semantic conventions、Langfuse / LangSmith / Arize、オブジェクトストレージ（S3等）、tail-based sampling、根拠リンク付き生成（grounded generation）、決定ログ（append-only）、不変監査ログ、モデル/プロンプトのバージョン記録。
