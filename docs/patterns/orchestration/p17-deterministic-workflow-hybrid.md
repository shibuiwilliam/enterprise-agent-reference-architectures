# P17 · DETERMINISTIC-WORKFLOW-HYBRID — エージェント＋決定論ワークフロー

!!! abstract "一言で"
    「考える（確率的・エージェント）」と「実行する（決定論的・ワークフロー）」を分離するパターン。エージェントは計画・抽出・判断を担い、不可逆な副作用は検証済みのワークフローエンジン/コードが実行する。確率的主体が不可逆操作を直接叩くリスクを構造的に排除する。不可逆操作の設計判断は人間のレビューが必須。

## 解決する問題

- **確率的主体が不可逆操作を直接叩く危険**: ハルシネーションによる誤った発注・送金・削除。
- **再現性の欠如**: エージェントの出力が毎回揺れるため、同じ入力でも異なる操作が実行される。
- **リトライ/ロールバック不能**: エージェントが直接APIを叩くと、途中失敗時の復旧が困難。
- **反復でのコスト・揺らぎ・遅延**: 同じパターンのタスクに毎回LLMを呼ぶのは非効率で出力が揺れる。

## 選定条件

**採用する場合（When to use）**

- 副作用が不可逆・高影響（発注/申請/プロビジョニング/支払い/契約変更）。
- 副作用が重い業務（発注・支払い・契約・プロビジョニング・本番変更）。
- 高頻度・形式安定・パターンが定まった反復タスク（定型レポート、繰り返しデータ抽出、定型WF、ETL的処理）で、LLM呼び出しを最小化したい。

**採用しない場合（When NOT）**

- 情報提供・要約のみの用途（ワークフロー分離のオーバーヘッドが不要）。
- 一回性・探索的・入力が毎回大きく異なるタスク（コンパイル型アプローチの効果が薄い）。

## 構造

- エージェントが構造化意図（パラメータセット）を出力する。
- 出力をJSONスキーマで検証し、形式の正しさを保証する。
- 検証済みパラメータを決定論ワークフロー（Temporal / Step Functions）に渡し、冪等・再試行可能な形で実行する。
- 失敗時はSaga（補償トランザクション）でロールバック可能にする。

### コンパイル型アプローチ（Plan-Once, Execute-Many）

決定論ワークフローの極端な形として、LLMを「コンパイラ」として使う手法がある。初回に意図を受け取り、検証済み実行計画（SQL / ワークフロー定義 / 抽出ルール / 関数）へ変換する。生成された計画を人間または eval で承認し、バージョン管理に登録する。以降の同種要求では、LLMを介さずに登録済み計画を決定論的に実行する。入力パターンが変化した場合は再コンパイルをトリガする。

このアプローチにより、反復実行時のコスト・揺らぎ・遅延・ハルシネーションを根絶できる。同一入力に対して常に同一の実行結果が得られ、監査にも耐える。

## 相反における立ち位置

不可逆・高影響・規制対象 → ワークフローに寄せ、エージェントは判断のみ。軽微・可逆・低頻度 → エージェントの自律比率を上げてよい。中間領域では、影響度と可逆性のマトリクスで判断し、段階的に自律度を上げる。

!!! note "相反する仕組みの選定（コンパイル時 vs ランタイムLLM）"
    高頻度・形式安定 → コンパイル型（Plan-Once, Execute-Many）。一回性・高変動 → ランタイムLLM。中間領域では、テンプレート化した計画に動的スロットだけLLMで埋める「ハイブリッド」が有効。判断基準は「同じパターンが何回繰り返されるか」と「入力の変動幅」。

## ユースケース

- Salesforceの営業エージェントが「見積書を作成して顧客に送付」する際、エージェントは見積内容（品目・金額・宛先）を構造化出力し、実際の見積書作成・PDF生成・メール送信はTemporalワークフローが実行する。
- Workdayの人事エージェントが「新入社員のプロビジョニング」を受けた際、必要な情報（氏名・部署・権限レベル）をエージェントが抽出し、Okta/Slack/Google Workspaceへのアカウント作成はStep Functionsが順序保証付きで実行する。
- Salesforceから毎週の営業レポートを生成するタスクで、初回にLLMがレポート生成クエリ（SOQL）と集計ロジックを作成。承認後は毎週同じクエリを自動実行し、LLMの呼び出しなしでレポートを生成する（コンパイル型）。
- ServiceNowの定型インシデント分類ルールを、LLMが一度だけ分類ロジック（決定木/ルールセット）に変換し、以降のチケットはルールベースで即時分類する（コンパイル型）。

## 実装メモ

Temporal、AWS Step Functions、JSONスキーマ検証、Saga（補償トランザクション）、冪等性キー、構造化出力（structured output）、DSL/クエリ生成、計画のバージョニング/キャッシュ。
