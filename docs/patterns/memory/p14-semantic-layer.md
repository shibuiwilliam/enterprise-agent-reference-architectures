# P14 · SEMANTIC-LAYER — セマンティックレイヤー / 組織知識グラフ

!!! abstract "一言で"
    指標・用語・組織・関係の意味定義を一元化し、エージェントの第一級コンテキスト源にするパターン。BIのセマンティックレイヤー（メトリクス定義）と組織知識グラフ（誰がどの部署/プロジェクト/権限か）を接続し、自然言語の曖昧性をハルシネーションではなく定義済みの事実で解決する。

## 解決する問題

- **数値/用語のハルシネーション**: 「売上」の定義が曖昧なまま回答すると、実際と異なる数値を生成する。
- **指示語の解決**: 「私のチーム」「先月」等の曖昧な参照を、組織構造・カレンダーに基づいて正確に解決できない。
- **組織階層スコープ**: 部署・プロジェクトに応じたデータ範囲の制御ができない。

## 選定条件

**採用する場合（When to use）**

- 「売上」「私のチーム」等の指標/用語/組織の曖昧性を解消したい／組織階層に沿ったスコープ制御が必要。
- 分析支援・組織横断の業務・権限依存の処理、指標や用語の定義が重要な業務。

**採用しない場合（When NOT）**

- 定義が固まっていない探索的な領域（定義の整備が先）。

## 構造

- BIのセマンティックレイヤー（dbt Semantic Layer / Cube 等）でメトリクス定義（「売上」=受注金額の税抜合計、期間はFY基準など）を一元管理。
- 組織グラフ（誰がどの部署・プロジェクト・役職・権限か）をSCIM/HRIS（Workday等）から同期。
- 自然言語→定義済みメトリクス/関係へのマッピングにより、ハルシネーションを抑制。「私のチームの売上」→「ユーザーの所属部署メンバーの受注金額合計」と解決。

## ユースケース

- Salesforce の営業分析エージェントが「先月の私のチームの売上を教えて」と問われた際、Workdayの組織グラフでチームメンバーを特定し、dbt Semantic Layerの「売上」定義に基づいて正確な数値を返す。
- ServiceNow のIT管理エージェントが「このプロジェクトの未解決チケット数」を聞かれた際、Linear/Jira連携のプロジェクトグラフでスコープを特定し、定義済みの「未解決」ステータス条件で集計する。

## 実装メモ

dbt Semantic Layer、Cube、知識グラフ（Neo4j / Amazon Neptune）、SCIM同期の組織データ、Workday/Okta連携。
