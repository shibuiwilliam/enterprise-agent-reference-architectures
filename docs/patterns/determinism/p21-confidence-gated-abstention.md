# P21 · CONFIDENCE-GATED-ABSTENTION — 信頼度ゲート棄権・エスカレーション

!!! abstract "一言で"
    エージェントが自信のないときに「答えない」ことを一級の出力として扱うパターンです。沈黙・棄権・人間へのエスカレーションを正常な動作として設計し、無理に答えてハルシネーションする最悪の挙動を防ぎます。

## 解決する問題

- **無理に答えてハルシネーションする最悪挙動**: 分からないことを分からないと言えず、誤情報を提供してしまいます。
- **過信**: エージェントが自分の出力に根拠のない確信を持ちます。
- **不適切な自動回答**: 専門知識が必要な質問や、前例のない状況で不正確な回答をしてしまいます。

## 選定条件

**採用する場合（When to use）**

- 誤答コストが高い（顧客対応/専門領域/リスクの高い助言）場合。
- 顧客対応・専門領域・リスクの高い助言など、誤答コストが高い業務。

**採用しない場合（When NOT）**

- 誤りが無害なブレインストーミングや探索的な対話。

## 構造

- 自己評価・検索ヒット品質・検証器合否（[P20](p20-dual-track-verification.md)）・不確実性シグナルの複合でスコアリングします。
- 閾値未満の場合は「分かりません」＋人間へ転送（warm handoff）を実行します。転送時は蓄積した文脈を引き継ぎ、人間がゼロから対応する必要をなくします。
- 顧客サポートではZendeskの有人席へ文脈ごとエスカレーションします。
- 棄権率・誤答率を継続的に計測し、閾値を調整します。

## 駆動変数とチューニング（程度）

閾値が高すぎると人間への丸投げが増え、エージェントの価値が消失します。低すぎると誤答が増加します。封じ込め率（エージェントが自力解決する割合）と誤答率のトレードオフ曲線を実測し、誤答コストが高い業務ほど棄権寄りに設定しましょう。起点として「検索ヒットの関連度スコア上位が閾値未満」「自己評価が低い」の条件を設定し、運用データで調整してください。

## ユースケース

- Zendesk の顧客サポートエージェントが、ナレッジベースに該当情報がない質問を受けた際、「この件についてはより詳しい担当者におつなぎします」と回答し、それまでの会話文脈をZendeskの有人チケットに転送します。
- Salesforce の営業エージェントが契約条件に関する質問を受けた際、法務的な判断が必要と検出し、「法務チームへの確認を推奨します」と棄権してSlackで法務チームに通知します。

## 実装メモ

不確実性推定、検索品質スコア、warm handoff（文脈付きエスカレーション）、エスカレーション経路（Zendesk / Slack / PagerDuty）。
