# P19 · COMPILED-AGENT — コンパイル型エージェント（Plan-Once, Execute-Many）

!!! abstract "メタ"
    **カテゴリ**: 確率→決定論 ／ **対象**: 両方 ／ **成熟度**: 高度 ／ **関連**: [P20](p20-dual-track-verification.md), [P24](../reliability/p24-semantic-cache.md)

**トリガ（採用検討の条件）**: 高頻度・パターンが安定・形式が定まった反復タスクがある。

## 概要

LLMを毎回呼ばず、一度だけ呼んで再利用可能な成果物（プラン/クエリ/DSL/コード）に変換（=コンパイル）し、以降は決定論的に実行するパターン。LLMをコンパイラとして使い、反復実行時のコスト・揺らぎ・遅延・ハルシネーションを根絶する。

## 設計

- 初回に意図を受け取り、検証済み実行計画（SQL / ワークフロー定義 / 抽出ルール / 関数）へ変換する。
- 生成された計画を人間または eval で承認し、バージョン管理に登録する。
- 以降の同種要求では、LLMを介さずに登録済み計画を決定論的に実行する。
- 入力パターンが変化した場合は再コンパイルをトリガする。

## 解決する課題

- **反復でのコスト・揺らぎ・遅延**: 同じパターンのタスクに毎回LLMを呼ぶのは非効率で出力が揺れる。
- **ハルシネーションの根絶**: 一度検証された計画を再利用するため、実行時のハルシネーションがない。
- **再現性と監査性**: 同一入力に対して常に同一の実行結果が得られ、監査に耐える。

## ユースケース

- Salesforceから毎週の営業レポートを生成するタスクで、初回にLLMがレポート生成クエリ（SOQL）と集計ロジックを作成。承認後は毎週同じクエリを自動実行し、LLMの呼び出しなしでレポートを生成する。
- ServiceNowの定型インシデント分類ルールを、LLMが一度だけ分類ロジック（決定木/ルールセット）に変換し、以降のチケットはルールベースで即時分類する。

## 向き / 不向き

- **向き**: 高頻度・形式安定・パターンが定まった反復タスク（定型レポート、繰り返しデータ抽出、定型WF、ETL的処理）。
- **不向き**: 一回性・探索的・入力が毎回大きく異なるタスク。

## 要素技術

構造化出力（structured output）、DSL/クエリ生成、計画のバージョニング/キャッシュ。

!!! note "相反する仕組みの選定（コンパイル時 vs ランタイムLLM）"
    高頻度・形式安定 → コンパイル型（P19）。一回性・高変動 → ランタイムLLM。中間領域では、テンプレート化した計画に動的スロットだけLLMで埋める「ハイブリッド」が有効。判断基準は「同じパターンが何回繰り返されるか」と「入力の変動幅」。
