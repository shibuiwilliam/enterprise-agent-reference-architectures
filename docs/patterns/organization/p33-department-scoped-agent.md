# P33 · DEPARTMENT-SCOPED-AGENT — 組織スコープ・エージェント

!!! abstract "一言で"
    組織の境界に沿ってエージェントのスコープを定義するパターン。**部署スコープ**（恒常的）では営業・IT・人事・法務など各部署固有の知識・ツール・権限・トーンを持つエージェントを配置し、**プロジェクトスコープ**（一時的）ではプロジェクトやタスクフォースの開始時にエージェント空間を払い出し、終了時にアーカイブして知識を組織知識へ昇華する。

## 解決する問題

- **権限/知識の自然な境界**: 部署ごとに異なるデータアクセス権限・業務知識を反映できない。
- **説明責任の明確化**: エージェントの所有者（部署長・プロジェクトオーナー）と責任範囲が不明確。
- **部署特有の文脈**: 営業用語・IT用語・人事ポリシーなど部署固有の知識が混在する。
- **チーム横断の文脈共有**: プロジェクトメンバー間で散在する情報を一箇所に集約できない。
- **プロジェクト知識の散逸**: プロジェクト終了とともにSlack・ドキュメント・チケットに散在した知識がアクセス困難になる。

## 選定条件

**部署スコープを採用する場合**

- 部署ごとに業務・ツール・規制が大きく異なる大企業。
- 恒常的な組織構造に沿った権限・責任の分離が必要。

**プロジェクトスコープを採用する場合**

- プロジェクト型業務がある／期間限定のタスクフォースやチーム横断の共同作業が必要。
- メンバーの参加・離脱に応じたアクセス制御の動的管理が求められる。

**採用しない場合（When NOT）**

- 横断業務が大半で部署境界が意味をなさない組織（部署スコープ）。
- 恒常的な定型業務のみ（プロジェクトスコープ）。

## 構造

### 部署スコープ（恒常的）

- 組織グラフと権限をSCIM/HRIS（Workday / Okta）から同期し、部署メンバーシップに基づくスコープを動的に維持する（[P14](../memory/p14-semantic-layer.md)）。
- 各部署エージェントの利用可能ツールとデータ範囲を部署単位で制約する。
- スーパーバイザ/ルーター（[P16](../orchestration/p16-supervisor-router.md)）が入口で意図を分類し、適切な部署エージェントへ振り分ける。
- エージェントのライフサイクルは組織変更に追従する（[P36](../governance/p36-digital-employee-lifecycle.md)）。

### プロジェクトスコープ（一時的）

- プロジェクト作成時にエージェント空間を自動プロビジョニング（Jira / Linear / Asana のプロジェクトに紐付け）。
- 参加メンバーのみアクセス可能（ReBAC でメンバーシップを制御）。
- サブプロジェクトは親プロジェクトの文脈を継承しつつ独立スコープを持つ。
- プロジェクト終了時にエージェント空間をアーカイブし、有用な知識はセマンティックメモリ（[P12](../memory/p12-layered-memory.md)）や組織知識グラフ（[P14](../memory/p14-semantic-layer.md)）へ昇華する。

## ユースケース

- 営業部エージェント（Salesforce・Slack連携、商談データへのアクセス権限）、IT部エージェント（ServiceNow・Jira連携、インフラデータへの権限）、人事部エージェント（Workday連携、給与・人事データへの限定権限）をそれぞれ独立して運用する。
- Slackの全社AIハブで「有給の残日数を教えて」→人事エージェント、「VPNが繋がらない」→ITエージェントへ自動振り分け。
- Jira上で新規プロジェクトが作成されると、プロジェクト専用のエージェント空間を自動生成。チケット・Confluenceドキュメント・Slackチャネルの情報を集約し、「このプロジェクトの未解決課題は？」「先週の進捗サマリ」等のクエリに対応する。
- 期間限定の障害対応タスクフォース用エージェントを払い出し、調査ログ・原因分析・対応状況を集約。対応完了後にポストモーテムとして知識をアーカイブ・組織知識へ還元する。

## 実装メモ

SCIM同期、HRIS連携（Workday / BambooHR）、組織グラフ（[P14](../memory/p14-semantic-layer.md)）、スーパーバイザ/ルーター（[P16](../orchestration/p16-supervisor-router.md)）、プロジェクト管理SaaS連携（Jira / Linear / Asana）、スコープ付きメモリ空間（[P12](../memory/p12-layered-memory.md)）、ReBAC（関係ベースアクセス制御）、ライフサイクル管理。
