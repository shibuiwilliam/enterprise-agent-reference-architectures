# P06 · AGENT-HUB — エージェント・ハブ

!!! abstract "メタ"
    **カテゴリ**: 統合 ／ **対象**: 両方 ／ **成熟度**: 標準 ／ **関連**: [P05](p05-in-app-copilot.md), [P16](../orchestration/p16-supervisor-router.md), [P08](../identity/p08-token-exchange-obo.md)

**トリガ（採用検討の条件）**: 横断業務が多い／ツールの発見性が課題／入口の一本化が価値をもたらす。

## 概要

社内のAI窓口を1つ用意し、全業務にルーティングするパターン。Slack上の単一ボットや専用Webアプリとして提供し、ユーザーは「どのツール・どのエージェントを使えばいいか」を意識せずに業務を依頼できる。[P05](p05-in-app-copilot.md) の対極にあたるアプローチ。

## 設計

- ユーザーの入力を意図分類し、適切なドメインエージェント（営業/IT/人事/開発 等）やツールへ委譲する。分類は [P16](../orchestration/p16-supervisor-router.md) のスーパーバイザ/ルーターが担う。
- 社内IdPでSSO認証後、各システムへはトークン交換（[P08](../identity/p08-token-exchange-obo.md)）でユーザー権限を伝播。
- 曖昧な依頼には聞き返しを行い、意図を確定してから委譲する。

## 解決する課題

- **「どのツールを使えばいいか分からない」問題**: ユーザーが業務内容を自然言語で伝えるだけで適切なエージェントに到達できる。
- **入口の一本化**: 全社のAI機能を単一エントリポイントに集約し、発見性を向上させる。
- **横断業務**: 複数システムにまたがるタスクを一箇所から実行できる。

## ユースケース

- Slack上の全社AIボットに「新入社員の入社手続きを進めて」と依頼すると、人事エージェントがWorkday・Okta・Slack・Google Workspace の設定を横断的に支援する。
- 専用Webポータルで「先月の営業パイプラインの状況を教えて」と問い合わせると、Salesforce・Slack・Jira のデータを横断して営業エージェントがサマリを提示する。

## 向き / 不向き

- **向き**: 横断業務が多い組織、全社展開、ツールの発見性向上が価値をもたらす場面。
- **不向き**: 深い文脈が要る専門作業（特定アプリ内で完結したほうが精度が高い場合は [P05](p05-in-app-copilot.md)）。

## 要素技術

Slack Bolt / Microsoft Teams アプリ、意図分類（[P16](../orchestration/p16-supervisor-router.md)）、トークン交換（[P08](../identity/p08-token-exchange-obo.md)）、フォールバック経路。

!!! note "相反する仕組みの選定（埋め込み vs ハブ）"
    単一システム完結・滞在時間が長い → 埋め込み（[P05](p05-in-app-copilot.md)）。横断業務・入口の一本化が価値 → ハブ（P06）。実務では併用が多く、ハブを正面玄関として全社に提供しつつ、高滞在アプリには個別に埋め込み、同じオーケストレーション層を共有する構成が現実的。
