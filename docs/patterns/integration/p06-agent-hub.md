# P06 · AGENT-HUB — エージェント・ハブ / 体験トポロジー

!!! abstract "一言で"
    エージェントをユーザーに届ける体験トポロジーを設計するパターンです。**ハブ型**は社内の単一AI窓口（Slack ボットや専用Webポータル）として全業務にルーティングし、ユーザーが「どのツール・どのエージェントを使えばいいか」を意識しなくてよい入口を提供します。**埋め込み型（コパイロット）**は既存システムのUI内（Salesforce のサイドパネル、Slack 内ボット、自社アプリのアシスタントウィジェットなど）にエージェントを配置し、コンテキストスイッチを排除します。実務では両方式を併用し、同じオーケストレーション層を共有する構成が現実的です。

## 解決する問題

- **「どのツールを使えばいいか分からない」問題（ハブ）**: ユーザーが業務内容を自然言語で伝えるだけで適切なエージェントに到達できます。
- **入口の一本化（ハブ）**: 全社のAI機能を単一エントリポイントに集約し、発見性を向上させます。
- **横断業務（ハブ）**: 複数システムにまたがるタスクを一箇所から実行できます。
- **コンテキストスイッチの削減（埋め込み）**: 別画面・別ツールへの遷移なしでAI支援を受けられます。
- **既存権限モデルの再利用（埋め込み）**: アプリの認証・認可をそのまま活用でき、追加の権限設計が不要です。
- **採用障壁の低さ（埋め込み）**: ユーザーが慣れた環境に自然に統合され、学習コストが低くなります。

## 選定条件

### ハブ型

**採用する場合（When to use）**

- 横断業務が多い／ツールの発見性が課題／入口の一本化が価値をもたらす。
- 全社展開でAI機能の発見性を高めたい場面。

**採用しない場合（When NOT）**

- 深い文脈が要る専門作業（特定アプリ内で完結したほうが精度が高い場合は埋め込み型が適切です）。

### 埋め込み型（コパイロット）

**採用する場合（When to use）**

- 作業が単一システム内で完結する／その画面に滞在時間が長い職種（営業・サポート・開発）がいる。

**採用しない場合（When NOT）**

- 本質的に複数システムの横断が必要な業務（ハブ型が適切です）。

## 構造

### ハブ型アーキテクチャ

- ユーザーの入力を意図分類し、適切なドメインエージェント（営業/IT/人事/開発 等）やツールへ委譲します。分類は [P16](../orchestration/p16-supervisor-router.md) のスーパーバイザ/ルーターが担います。
- 社内IdPでSSO認証後、各システムへはトークン交換（[P08](../identity/p08-token-exchange-obo.md)）でユーザー権限を伝播します。
- 曖昧な依頼には聞き返しを行い、意図を確定してから委譲します。

### 埋め込み型（コパイロット）アーキテクチャ

- 現画面のコンテキスト（表示中のレコード・選択範囲・ユーザーの役割）を構造化してエージェントに渡します。
- アクションはそのアプリのAPIに閉じ、認可は既存セッション（[P08](../identity/p08-token-exchange-obo.md)）を流用可能です。
- アプリ固有のデータモデルとワークフローに特化し、深い文脈理解と高精度な提案を実現します。

## 相反における立ち位置

!!! note "相反する仕組みの選定（埋め込み vs ハブ）"
    単一システム完結・滞在時間が長い → 埋め込み（コパイロット）。横断業務・入口の一本化が価値 → ハブ。実務では併用が多く、ハブを正面玄関として全社に提供しつつ、高滞在アプリには個別に埋め込み、同じオーケストレーション層を共有する構成が現実的です。

## ユースケース

- Slack上の全社AIボットに「新入社員の入社手続きを進めて」と依頼すると、人事エージェントがWorkday・Okta・Slack・Google Workspace の設定を横断的に支援します。（ハブ）
- 専用Webポータルで「先月の営業パイプラインの状況を教えて」と問い合わせると、Salesforce・Slack・Jira のデータを横断して営業エージェントがサマリを提示します。（ハブ）
- Salesforce 内で商談レコードを表示中に、次のアクション提案・メール下書き・類似案件検索をサイドパネルのコパイロットが提示します。（埋め込み）
- ServiceNow のインシデントチケット画面で、チケット分類・優先度判定・解決手順の提案をインラインで行います。（埋め込み）
- Slack 上で業務チャネルの文脈を読み取り、関連ドキュメントの検索やタスク作成を支援するボットです。（埋め込み）

## 実装メモ

- **ハブ型**: Slack Bolt / Microsoft Teams アプリ、意図分類（[P16](../orchestration/p16-supervisor-router.md)）、トークン交換（[P08](../identity/p08-token-exchange-obo.md)）、フォールバック経路。
- **埋め込み型**: Slack Bolt、Salesforce LWC（Lightning Web Components）、Microsoft Teams アプリ、Webサイドパネル/ウィジェット。
