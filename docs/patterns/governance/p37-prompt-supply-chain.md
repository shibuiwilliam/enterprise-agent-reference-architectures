# P37 · PROMPT-SUPPLY-CHAIN — プロンプト・コンテキストのサプライチェーン管理

!!! abstract "一言で"
    プロンプト・システム指示・テンプレートをコード成果物として扱い、バージョン管理・レビュー・署名・eval通過を経てデプロイするパターンです。ソフトウェアのCI/CDと同等の品質管理プロセスをプロンプトに適用します。

## 解決する問題

- **「こっそり変えて挙動が変わった」問題**: プロンプト変更の追跡・レビューができず、品質劣化の原因が特定できません。
- **テンプレート経由のインジェクション**: 外部入力がプロンプトテンプレートに注入され、意図しない動作を引き起こします。
- **再現性の喪失**: どのバージョンのプロンプトで生成された結果か追跡できません。

## 選定条件

**採用する場合（When to use）**

- 本番運用で複数人が開発に関わる／プロンプト変更で挙動が変わることを統制したい。
- 本番運用・複数人開発のエージェント。

**採用しない場合（When NOT）**

- 使い捨ての実験（統制のオーバーヘッドが見合わない）。

## 構造

- プロンプトをGit＋レジストリで管理し、変更はPR・コードレビューを必須とします。
- 変更時は eval gate（[P26](../reliability/p26-eval-gate-canary.md)）の通過を必須にします。
- 本番デプロイされたプロンプトには署名を付与し、改竄検知を行います。
- 外部入力の埋め込み箇所（テンプレート変数）には注入対策（区切り・サニタイズ・最小権限化）を施します。

## ユースケース

- Zendesk の顧客対応エージェントのプロンプトを改善する際、GitでPRを作成→チームレビュー→eval gate でゴールデンデータセットに対する品質チェック→承認後にデプロイの流れで管理します。
- 複数チームが共有するシステムプロンプトのテンプレートを署名付きでレジストリに格納し、改竄されていないことをデプロイ時に検証します。

## 実装メモ

プロンプトレジストリ、Gitワークフロー（PR/レビュー）、署名・改竄検知、テンプレートのサンドボックス化（[P11](../identity/p11-zero-trust-sandbox.md)）。
