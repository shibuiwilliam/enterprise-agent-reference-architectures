# 部門別適用例

エンタープライズAIエージェントが各部門で動かす**成果KPI**は部門ごとに大きく異なります。営業は受注率とパイプライン、人事は採用リードタイムと離職率、カスタマーサポートはCSATと一次解決率、エンジニアリングはリードタイムとMTTR、経営は意思決定速度と全社コスト最適化といった具合です。それぞれの価値ユースケースに対して、安全に実現するためのパターンの束が決まってきます。

各部門ページでは「どの成果KPIをどう動かすか」（価値ユースケース＋成果KPIマッピング＋価値の階段）を主軸に、その価値を安全に実現するためのパターン適用理由を解説します。

|---|---|---|---|
| [Sales Agent](sales.md) | 受注率向上・商談サイクル短縮・パイプライン健全化 | 受注率・案件サイクル・パイプライン金額・客単価 | ID-2, ID-4, IN-2, KM-5, RT-5, RT-4 |
| [HR Agent](hr.md) | 採用効率化・離職予防・問い合わせ自己解決 | 採用リードタイム・内定承諾率・離職率・自己解決率 | KM-4, KM-6, RT-4, GV-4, RT-6 |

## パターンを選ぶ視点

部門に合ったパターン束を選ぶときは、以下の6つの軸で考えると整理しやすくなります。

### 価値軸（どの成果KPIを動かすか）

**ビジネス価値タイプ**：売上貢献（Sales）・コスト最適化（HR/CS/Engineering）・意思決定加速（Executive）のどれが主眼かによって、価値ユースケースの選定が変わります。各部門ページの「価値ユースケース」表と「成果KPIマッピング」図で、具体的な因果経路を確認できます。

**価値実現の時間軸**：クイックウィン（読み取り専用、1〜4週間で効果実感）→ 分析・示唆（1〜3ヶ月）→ 自動化（3〜12ヶ月）という段階設計は全部門共通です。各部門ページの「価値の階段」で段階ごとの機能と期待成果を示しています。詳細は[価値成熟度ロードマップ](../value-maturity-roadmap.md)を参照してください。

### 安全軸（価値を安全に実現するために）

**データ機密度**：給与・評価・経営情報など個人情報保護法・会社法上のリスクが高いデータを扱う部門ほど、DLP・スコープ分離・監査のパターンを厚くする必要があります。

**書き込みリスク**：読み取りだけでなく CRM・ERP・HR システムへの書き込みを行う場合は、Command Envelope（RT-5）や SoR Write Boundary（RT-6）で操作を構造化し、Human Approval（RT-4）で承認を挟みます。

**顧客接点の有無**：顧客と直接やり取りするエージェント（カスタマーサポート等）は、社内従業員向けエージェントと ID・権限・ログを完全に分離します（ID-1）。面をまたぐ経路を構造的に排除することが最も重要な設計事項です。

**実行環境のリスク**：コード実行や本番インフラ操作を伴うエンジニアリング用途では、Zero-Trust PDP/PEP（ID-6）とサンドボックス（IN-1 相当）による強制隔離が前提です。
