# パターン記述フォーマット規約

各パターンページは固定フィールドを持つ。コーディングエージェントはこのキーで抽出してよい。
（執筆者向けの詳細な規約は `CLAUDE.md` を参照）

| フィールド | 意味 |
|---|---|
| フロントマター | `id` / `slug` / `title` / `category` / `audience` / `maturity` / `related` / `status`。`id`・`slug` は不変。 |
| メタ行（abstract admonition） | カテゴリ / 対象（従業員・顧客・両方）/ 成熟度（基盤・標準・高度）/ 関連パターン |
| トリガ | 採用を検討すべき要件条件。**要件と機械照合できるIF句**として記述する。 |
| 概要 | 何をするパターンか |
| 設計 | 構成・主要コンポーネント・データフロー |
| 解決する課題 | 適用しないと起きる問題 |
| ユースケース | 前提企業のSaaSに即した具体例 |
| 向き / 不向き | 適合・非適合の条件 |
| 要素技術 | 代表的な実装技術 |
| ⚠️ 程度の判断 | 該当時のみ。`!!! warning` で記述（パラメータのチューニング軸） |
| ⚖️ 相反の選定 | 該当時のみ。`!!! note` で記述（二者択一の判断基準） |

## フロントマターの値

- `audience`: `both`（両方）/ `employee`（従業員向け）/ `customer`（顧客向け）
- `maturity`: `基盤` / `標準` / `高度`
- `status`: `draft`（スタブ）/ `review`（要レビュー）/ `done`（完成）
- `related`: 関連パターンのID配列（例 `[P08, P09]`）。機械可読・不変。

## 最小テンプレート

````markdown
---
id: P03
slug: AI-GATEWAY
title: "P03 · AI-GATEWAY — AIエージェント・ゲートウェイ"
category: 統合
audience: both
maturity: 基盤
related: [P08, P09, P23, P24, P25, P27]
status: draft
---
````
