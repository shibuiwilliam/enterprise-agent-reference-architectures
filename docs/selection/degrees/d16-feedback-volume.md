# D16 · フィードバック収集量

!!! abstract "一言で"
    エージェントの応答に対するフィードバック（評価・修正・コメント）をどの程度
    収集するかの程度。多く集めれば改善材料は増えるが、レビュー・分析の負荷も増す。
    低評価・高コスト・新パターンのケースを優先的に抽出するのが効率的。

## トレードオフ

| 緩める方向（多く収集） | 締める方向（少なく収集） |
|---|---|
| 改善のための材料が豊富になる。パターンの発見・品質傾向の把握が容易 | レビュー・ラベリングの人的コストを抑えられる |
| 収集したフィードバックの消化が追いつかず、未処理の山ができる | サンプル不足で統計的に有意な傾向を把握できない |
| ユーザーに頻繁にフィードバックを求め、体験を阻害する | 重大な品質問題の検出が遅れる |

## 判断軸と決め方

### 優先抽出の原則

全量収集が理想だが、レビューリソースには限りがある。以下の優先順位で抽出する。

1. **低評価のケース（thumbs down / 1-2 star）**: 品質問題の直接的な証拠。全量をレビュー対象にする。
2. **高コストのケース（トークン消費・API 呼び出しが多い）**: 非効率なパスの改善余地を示す。上位 10% を抽出。
3. **新しいパターン（既存の eval でカバーされていないタスク）**: 未知の失敗モードの発見につながる。クラスタリングで検出。
4. **エスカレーションされたケース**: エージェントが棄権した案件。棄権が適切だったかを検証する。
5. **正常完了の高評価ケース**: 成功パターンの強化学習データとして、サンプリングで一定量確保する。

### 収集方法の設計

- **暗黙的フィードバック**: ユーザーの行動から自動収集。回答後の離脱率、修正回数、同じ質問の再投稿など。ユーザー体験を阻害しない。
- **明示的フィードバック（軽量）**: thumbs up/down の 2 択。回答の末尾に表示。収集率は目安として 10〜20%。
- **明示的フィードバック（詳細）**: 自由記述のコメント欄。低評価時のみ表示し、何が問題だったかを具体的に取得する。
- **専門家レビュー**: 定期的にサンプルを抽出し、ドメイン専門家が品質を評価する。コストは高いが精度が高い。

### エンタープライズでの典型シナリオ

- **Zendesk 顧客対応**: 全回答に thumbs up/down を付与。低評価は全量レビュー、高評価は 5% サンプリング。月次で顧客満足度との相関を分析。
- **Slack 社内ボット**: リアクション絵文字を暗黙的フィードバックとして収集。「ありがとう」系は高評価、「違う」系は低評価として分類。
- **Salesforce 営業支援**: エージェントの提案を営業が採用したかどうかを暗黙的フィードバックとして追跡。採用率が eval の指標になる。

## 目安（出発点）

| フィードバック種別 | 収集率の目安 | レビュー対象 |
|---|---|---|
| 暗黙的（行動ベース） | 全量自動収集 | 異常値のみアラート |
| 明示的（thumbs up/down） | 全回答に表示、回答率 10〜20% | 低評価は全量、高評価は 5% |
| 明示的（自由記述） | 低評価時のみ表示 | 全量 |
| 専門家レビュー | 月次 50〜100 件サンプリング | 全量 |

## 計測指標

- **フィードバック収集率**: フィードバックが得られた回答の割合。目安として明示的フィードバックは 10% 以上。
- **低評価率の推移**: 週次・月次での低評価の割合と傾向。上昇トレンドは品質劣化のシグナル。
- **フィードバック→改善のリードタイム**: 低評価を受けてから修正がデプロイされるまでの日数。
- **レビュー消化率**: 収集したフィードバックのうち、レビュー済みの割合。70% 未満なら収集量を絞るか、レビュー体制を強化する。
- **改善効果の定量化**: フィードバックに基づく修正の前後で、同種のタスクの品質指標がどう変化したか。

## 調整方針

1. **レビュー消化率が 70% を下回ったら** 収集対象を優先度の高いカテゴリに絞る。未消化のフィードバックは価値を生まない。
2. **低評価率が安定して低い（2% 以下）場合**、明示的フィードバックの表示頻度を下げ、ユーザー体験を優先する。
3. **新機能・新ツールのリリース直後** は、一時的に収集率を上げて初期品質を集中的に監視する。
4. **フィードバックの分析は自動化** する。クラスタリングで頻出パターンを検出し、人間は判断と対応策の決定に集中する。
5. **フィードバックの偏りに注意** する。積極的にフィードバックするユーザーの意見が過剰に反映されないよう、ユーザー属性で層化サンプリングする。
6. **プライバシーに配慮** し、フィードバックに含まれる個人情報を匿名化した上で分析・保存する。
