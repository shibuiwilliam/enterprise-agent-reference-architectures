# D07 · HITL対象範囲

!!! abstract "一言で"
    AIエージェントの出力・アクションのうち、人間の承認（Human-in-the-Loop）を必須とする
    範囲を制御するチューニングパラメータです。広すぎると承認待ちがボトルネックになり自動化の
    恩恵が失われ、狭すぎると重大なミスが人間のチェックなしに実行されるリスクが生じます。

## トレードオフ

| 緩める方向（広い HITL 対象） | 締める方向（狭い HITL 対象） |
|---|---|
| 重大な判断ミスを人間が事前に防止でき、安全性が高い | エージェントのスループットが向上し、自動化の効果が最大化される |
| 説明責任を明確に果たせる（「人間が承認した」記録が残る） | 承認待ち渋滞が発生せず、処理遅延が解消される |
| 初期導入時のリスク軽減と信頼構築に有効 | 承認の形骸化（内容を確認せずに承認するだけ）を防げる |

## 判断軸と決め方

HITL の対象範囲は**影響度 x 可逆性 x 実績精度**の3軸で判断します。

**影響度（Impact）**

- **高影響**: 金額が大きい（Workday の経費承認で目安として 10 万円超）、外部への公開を伴う（顧客へのメール送信）、法的拘束力がある（契約条件の変更）→ HITL 必須
- **中影響**: 社内チケットの更新（Jira/ServiceNow）、社内 Slack チャンネルへの投稿 → エージェントの実績に応じて段階的に自動化
- **低影響**: 情報の検索・要約、下書き生成、社内ナレッジの参照 → HITL 不要

**可逆性（Reversibility）**

- **不可逆**: メール送信、外部 API への書き込み、顧客への通知 → HITL の対象に含めます
- **可逆**: 下書き保存、社内チケットのステータス変更、スプレッドシートの更新 → HITL から除外可能です

**実績精度（Track Record）**

エージェントの過去の判断精度を実測し、精度が十分に高い操作から段階的に HITL を解除します。

- **導入初期（〜1ヶ月）**: 高影響・中影響の操作はすべて HITL
- **安定期（1〜3ヶ月）**: 中影響で精度 95% 超の操作を自動化候補にします
- **成熟期（3ヶ月〜）**: 高影響でも定型化が進んだ操作は、サンプリング監査付きの自動化を検討します

## 目安（出発点）

- 金額影響あり（目安として 10 万円超）: HITL 必須
- 外部顧客への直接送信: HITL 必須
- 社内システムへの書き込み: 導入初期は HITL、精度実績に応じて段階解除
- 情報検索・要約・下書き: HITL 不要
- 承認待ちタイムアウト: 目安として 4 時間（超過時はエスカレーション）

## 計測指標

- **承認待ち時間（中央値・P95）**: 承認リクエストからの応答時間。長すぎると自動化の効果が失われます
- **承認却下率**: 人間が承認せず修正・却下した割合。低すぎる場合は形骸化の兆候です
- **ラバースタンプ率**: 目安として 5 秒以内に承認されたケースの割合。内容を確認していない可能性があります
- **HITL 解除後のインシデント率**: 自動化に移行した操作で問題が発生した割合

## 調整方針

- 承認待ち時間の P95 が目安として 2 時間を超える場合: HITL 対象を縮小するか、承認者の増員・ルーティングを改善してください
- 承認却下率が 1% 未満かつラバースタンプ率が 80% 超の場合: HITL が形骸化しています。対象操作の自動化を検討してください
- HITL 解除後にインシデントが発生した場合: 該当操作を HITL に戻し、原因分析後に改善してから再度解除を検討してください
