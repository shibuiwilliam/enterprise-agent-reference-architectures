    成熟度ロードマップ¶

    エンタープライズ AI エージェントの導入は段階的に進める。以下の5フェーズで成熟度を高める。

    flowchart LR
    P1[Phase 1<br/>Read-only<br/>Copilot] --> P2[Phase 2<br/>Department<br/>Assistant]
    P2 --> P3[Phase 3<br/>Workflow<br/>Agent]
    P3 --> P4[Phase 4<br/>Enterprise<br/>Platform]
    P4 --> P5[Phase 5<br/>AI-native<br/>Operating Model]

    Phase 1：Read-only Enterprise Copilot¶

    社内文書検索・要約・FAQ・個人支援。まずシャドー AI を止めて小さく試す。

    パターン 役割
    KM-1 権限認識 RAG 権限付き文書検索
    OB-1 Observability Lake トレース基盤
    GV-5 Model Gateway LLM 呼び出し統制
    GV-1 Control Plane エージェント登録

    Phase 2：Department Assistive Agent¶

    部門テンプレートで SaaS 連携エージェントを量産。下書き・チケット案・CRM 更新案を提案し、承認付きで実行する。

    パターン 役割
    GV-3 Department Agent Factory 部門テンプレート量産
    IN-2 SaaS Adapter SaaS 差吸収
    RT-4 Human Approval 承認付き実行
    ID-2 OBO 本人権限での SaaS アクセス

    Phase 3：Workflow Agent¶

    長尺の業務プロセスを自動化。承認待ち・障害からの再開、複数システム更新の Saga、イベント駆動の自律処理を導入する。

    パターン 役割
    RT-8 Durable Workflow 永続ワークフロー
    RT-7 Saga 補償トランザクション
    RT-10 Event-Driven イベント駆動
    RT-9 Work Queue 業務キュー参加

    Phase 4：Enterprise Agent Platform¶

    全社基盤として統治・認可・コスト・評価・事故対応を整備する。

    パターン 役割
    GV-1 Control Plane 全社統制
    IN-1 Tool/MCP Gateway ツール統制
    ID-7 Policy-as-Code 決定論的認可
    KM-4 Scoped Memory メモリ統治
    GV-8 Cost Quota コスト配賦
    GV-7 Eval Pipeline 評価 CI/CD
    GV-9 Incident Response 事故対応
    ID-4 Permission Mirror 権限同期
    KM-3 Canonical Object 組織グラフ

    Phase 5：AI-native Operating Model¶

    人間と AI の業務分担を再設計し、数万人規模の容量・コスト・可用性を最適化する。

    パターン 役割
    GV-3 Department Factory 全部門展開
    RT-11 Project Twin チームの AI メンバー
    RT-2 RACI Multi-Agent 責任分担マルチ
    GV-4 Industry Policy Pack 業界別統制
    ID-1 二面分離 顧客面展開
    GV-10 Value Measurement 二層価値計測
